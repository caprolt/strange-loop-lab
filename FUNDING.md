# Funding

Strange Loop Lab is intended to be a public-interest project. This funding policy explains how the project thinks about money, sponsorship, independence, and conflicts of interest.

The project is currently an independent open-source project, not a registered nonprofit. If that status changes, this document will be updated.

## Funding principles

### 1. Independence comes first

Funding must not control findings, scores, rankings, publication decisions, or experiment design in ways that compromise the project.

### 2. No pay-to-win benchmarks

The project will not sell favorable treatment.

This means no:

- paid favorable rankings
- hidden benchmark placement
- sponsor-approved conclusions
- suppression of negative results
- paid removal of findings
- misleading certification badges
- private scoring rules for sponsors only

### 3. Disclose material support

Material support should be disclosed near the work it affects.

This includes:

- grants
- sponsorships
- donations
- API credits
- compute credits
- vendor-provided access
- paid consulting relationships
- affiliate relationships
- employment or contractor relationships

### 4. Keep public work public

The core project should remain publicly inspectable.

Private work may exist in limited cases, such as responsible disclosure, safety review, or pre-publication analysis, but the project should not drift into closed vendor consulting as its main identity.

### 5. Avoid dependency on a single funder

A public-interest project is fragile if one funder can determine whether it survives.

The project should prefer diversified support over dependency on one company, donor, or institution.

## Current funding status

Current status:

> Strange Loop Lab is not currently funded by sponsors, grants, donations, or a nonprofit entity unless otherwise stated in the repository.

Maintainers should update this section if material funding is received.

## Acceptable funding sources

The project may consider support from:

- individual donations
- public-interest grants
- research grants
- philanthropic support
- nonprofit fiscal sponsorship
- unrestricted sponsorships
- donated API or compute credits
- paid work that does not compromise public findings

## Funding sources requiring caution

The project should use extra caution with funding from:

- AI model providers evaluated by the project
- companies selling AI safety or eval products
- companies using project outputs for marketing
- political organizations
- advocacy groups seeking predetermined conclusions
- organizations requesting private influence over public methodology
- funders requiring review or approval before publication

Caution does not always mean refusal. It means disclosure, boundaries, and review.

## Funding the project may reject

The project should reject funding that requires or strongly pressures the project to:

- change findings for the funder
- hide unfavorable results
- create promotional reports disguised as independent research
- attack a competitor without fair methodology
- publish unsafe details
- collect private user data unnecessarily
- endorse a model, vendor, candidate, or product beyond the evidence
- limit publication of public-interest findings
- misrepresent the project's nonprofit or legal status

## Sponsorship policy

Sponsors may be acknowledged if the support is real and disclosed.

Sponsor acknowledgment does not imply endorsement.

Sponsor acknowledgment should not appear in a way that suggests the sponsor controls research outcomes.

Suggested language:

> "This work received unrestricted support from [Sponsor]. The sponsor did not control experiment design, scoring, conclusions, or publication decisions."

Avoid language like:

> "Approved by [Sponsor]."

or

> "Certified safe by Strange Loop Lab."

unless a formal certification program exists, which it currently does not.

## API and compute credits

API or compute credits can create subtle bias because they make some models easier or cheaper to test.

When credits are used, disclose them.

Example:

> "Runs against Model A used provider-donated API credits. The provider did not review or approve the report before publication."

## Paid consulting and private work

Maintainers or contributors may do paid consulting outside the project. Relevant conflicts should be disclosed when they touch project work.

If the project itself takes on paid work, the terms should preserve independence.

Paid work should not require hidden changes to public methodology or conclusions.

## Donations

If donations are accepted before formal nonprofit status or fiscal sponsorship exists, the project should be clear that donations may not be tax-deductible.

Do not imply 501(c)(3) status unless it has been legally granted.

## Fiscal sponsorship

The project may seek fiscal sponsorship if public donations, grants, or shared project funds become important.

Fiscal sponsorship may help with:

- donation processing
- accounting
- compliance
- grant eligibility
- public trust
- separation between personal and project funds

If fiscal sponsorship is established, this file should name the fiscal sponsor and link to the relevant terms.

## Future nonprofit status

The project may eventually become a standalone nonprofit if the work grows enough to justify the administrative responsibility.

Possible reasons to form a nonprofit include:

- recurring donations
- grant opportunities
- institutional partnerships
- multiple maintainers or contributors
- need to hold project assets independently
- need for formal governance
- public-interest credibility

Until then, the priority is to produce useful, reproducible work.

## Expense transparency

If the project receives meaningful funding, it should publish periodic summaries of how funds are used.

Possible categories:

- API costs
- compute costs
- hosting
- domain names
- software tools
- research support
- documentation
- legal or accounting help
- accessibility improvements
- contributor stipends

## Contributor compensation

The project may eventually pay contributors if funding allows.

Paid contributor work should be disclosed when it materially affects a report or experiment.

Payment should not determine findings.

## No investment framing

Strange Loop Lab is not intended to be a venture-backed startup or growth product.

The project may build software, publish research, accept funding, or form an institution. But the core purpose is public understanding and accountability, not extraction.

## Funding updates

Funding disclosures should be updated whenever material support changes.

If in doubt, disclose.
